Diagnosis

- Relevant error lines: [line 19](#L19), the conflict markers around the lockfile portion at [line 22](#L22)-[line 24](#L24) and repeated at [line 31](#L31)-[line 34](#L34), and the final failure lines [line 45](#L45)-[line 46](#L46).
- Error type and cause: pnpm failed while parsing a YAML lockfile because it contains unresolved Git merge conflict markers ("<<<<<<< HEAD", "=======", ">>>>>>>"). The YAML parser treats these markers as invalid syntax and aborts install (see the "can not read a block mapping entry" parse error at [line 19](#L19) triggered by the conflict marker at [line 23](#L23)).

Solution

1) Inspect and resolve the lockfile conflict
- The problematic file is the repository lockfile (pnpm-lock.yaml). Open pnpm-lock.yaml and remove the Git conflict markers, resolving which side of the conflict to keep for each section. If you prefer, use a merge tool to choose the correct content.
- Example manual steps:
```bash
# show files with conflicts
git status

# open the lockfile and edit pnpm-lock.yaml to remove <<<<<<<, =======, >>>>>>> markers
# After editing:
git add pnpm-lock.yaml
git commit -m "Resolve pnpm-lock.yaml merge conflict"
git push
```

2) Alternatively regenerate the lockfile (if you prefer to replace it)
- Use pnpm to regenerate the lockfile (recommended if your project uses pnpm). This will create a clean pnpm-lock.yaml without conflict markers.
```bash
git checkout -- pnpm-lock.yaml   # if you want to discard local changes first (optional)
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
git add pnpm-lock.yaml
git commit -m "Regenerate pnpm-lock.yaml"
git push
```
- If your project standard is npm, ensure you and CI use the same package manager. If you remove pnpm-lock.yaml, be sure your team agrees and CI/build commands are updated accordingly.

3) Re-run the Netlify build
- After committing the resolved or regenerated pnpm-lock.yaml, push to the branch Netlify builds from. The new build should proceed past dependency installation.

Notes and checks
- If you see merge conflicts locally (git status will show unmerged paths), resolve them before pushing.
- If you want me to point to the exact lockfile in your repo, I can (e.g. pnpm-lock.yaml at the repo root).
